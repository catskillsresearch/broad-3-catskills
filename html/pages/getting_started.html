Installation

This section guides you through setting up the environment and installing the required dependencies for the Catskills Solution.

Requirements

The solution requires the following:

  • Python 3.8 or higher
  • CUDA-compatible GPU (recommended for full dataset)
  • At least 16GB RAM (32GB recommended for full dataset)
  • At least 100GB disk space for the full dataset and outputs

Environment Setup

We recommend using a virtual environment:

Setting Up Environment 

# Clone the repository
git clone https://github.com/catskills/ibd-challenge-crunch3.git
cd ibd-challenge-crunch3

# Create and activate virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

Weights & Biases Setup

Our solution uses Weights & Biases for hyperparameter optimization and experiment tracking. You'll need to set up an account and log in:

Weights & Biases Setup 

# Install wandb if not already installed
pip install wandb

# Log in to wandb
wandb login

Follow the prompts to complete the login process. You'll need to create an account at wandb.ai if you don't already have one.

Data Setup

Before running the pipeline, you need to prepare the data in the streamlined structure.

Data Preparation

Run the data preparation script to organize the raw data into the streamlined structure:

Data Preparation 

python scripts/run_pipeline.py --config config/config.yaml --step data

This script will:

  • Create the streamlined data structure as described in the Data Structure page
  • Remove unused data elements (like unregistered images)
  • Organize the data by type (images, spatial, scRNA-seq, annotations)
  • Create the necessary directories for pipeline outputs

Data Structure Verification

After running the data preparation script, verify that the data structure matches the expected structure:


data/
├── images/
│   ├── HE_registered/
│   └── HE_nuc_registered/
├── spatial/
├── scrna/
├── annotations/
└── processed/

If any directories are missing or empty, check the logs for errors.

Small Dataset Pipeline

We strongly recommend starting with the small dataset pipeline to validate the approach and optimize hyperparameters before scaling to the full dataset.

Creating the Small Dataset

Create a small subset of the data for rapid experimentation:

Creating Small Dataset 

python scripts/create_small_dataset.py --data_dir data --output_dir data/processed/small_dataset --num_samples 2 --num_cells 1000 --num_genes 100

You can adjust the parameters to control the size of the small dataset:

  • --num_samples: Number of samples to include (default: 2)
  • --num_cells: Number of cells per sample (default: 1000)
  • --num_genes: Number of genes to include (default: 100)

A smaller dataset will run faster but may not capture all the complexity of the full dataset. We recommend starting with the default values and adjusting as needed.

Running the Small Dataset Pipeline

To run the complete pipeline on the small dataset:

Running Small Dataset Pipeline 

python scripts/run_pipeline.py --config config/config.yaml --small_dataset

This will run the entire pipeline on the small dataset, including:

  1. Feature extraction from the small dataset images
  2. Hyperparameter search on the small dataset
  3. Model training with the optimal hyperparameters
  4. Gene expression prediction
  5. Gene ranking

The results will be saved in the data/processed/ directory with a small_dataset prefix to distinguish them from full dataset results.

Hyperparameter Search

The hyperparameter search is a critical component of our approach, allowing us to find the optimal model configuration.

Running Hyperparameter Search on Small Dataset

To run just the hyperparameter search on the small dataset:

Running Hyperparameter Search on Small Dataset 

python scripts/run_hyperparameter_search.py --config config/hyperparameter_search.yaml --small_dataset

Verifying Hyperparameter Search

To verify that the hyperparameter search is working correctly and selecting optimal parameters, run it multiple times with different random seeds:

Running Multiple Hyperparameter Searches 

# Run with different random seeds
python scripts/run_hyperparameter_search.py --config config/hyperparameter_search.yaml --small_dataset --seed 42
python scripts/run_hyperparameter_search.py --config config/hyperparameter_search.yaml --small_dataset --seed 123
python scripts/run_hyperparameter_search.py --config config/hyperparameter_search.yaml --small_dataset --seed 456

Compare the results in the Weights & Biases dashboard to ensure that the search is consistently finding similar optimal parameters.

Analyzing Hyperparameter Search Results

The hyperparameter search results can be analyzed in the Weights & Biases dashboard:

  1. Open the Weights & Biases Dashboard

    Go to wandb.ai and log in to your account.

  2. Navigate to Your Project

    Find the project for your hyperparameter search (default: "ibd-challenge-crunch3").

  3. View the Sweep

    Click on the sweep to view the results of all runs.

  4. Analyze Parameter Importance

    Use the "Parameter Importance" tab to see which hyperparameters have the most impact on performance.

  5. Compare Runs

    Use the "Parallel Coordinates" view to compare different runs and understand the relationships between hyperparameters and performance.

The optimal hyperparameters will be automatically saved to data/processed/hyperparameters/optimal_params.json for use in the full pipeline.

Full Pipeline

Once you've verified that the small dataset pipeline is working correctly and found optimal hyperparameters, you can run the full pipeline on the complete dataset.

Running Hyperparameter Search on Full Dataset

If desired, you can refine the hyperparameters on the full dataset, starting from the optimal parameters found on the small dataset:

Running Hyperparameter Search on Full Dataset 

python scripts/run_hyperparameter_search.py --config config/hyperparameter_search.yaml

This will perform a more focused search around the optimal parameters found on the small dataset.

Running the Full Pipeline

To run the complete pipeline on the full dataset:

Running Full Pipeline 

python scripts/run_pipeline.py --config config/config.yaml

This will run the entire pipeline on the full dataset, using the optimal hyperparameters found during the hyperparameter search.

Running Individual Pipeline Steps

You can also run individual steps of the pipeline:

Running Individual Pipeline Steps 

# Extract features
python scripts/run_pipeline.py --task ExtractFeatures --config config/config.yaml

# Train model
python scripts/run_pipeline.py --task TrainModel --config config/config.yaml

# Predict genes
python scripts/run_pipeline.py --task PredictGenes --config config/config.yaml

# Rank genes
python scripts/run_pipeline.py --task RankGenes --config config/config.yaml

Luigi will automatically run any dependencies that haven't been completed yet.

Monitoring Pipeline Progress

You can monitor the progress of the pipeline using the Luigi web interface:

Starting Luigi Web Interface 

# In a separate terminal
luigid

Then open http://localhost:8082 in your web browser to view the pipeline status.

Troubleshooting

Here are some common issues and their solutions:

Data Preparation Fails

If the data preparation script fails, check the following:

  • Ensure the raw data directory exists and contains the expected files
  • Check that you have sufficient disk space
  • Verify that you have write permissions for the output directory
  • Check the logs for specific error messages
Out of Memory Errors

If you encounter out of memory errors:

  • Reduce the batch size in the configuration file
  • Use a smaller small dataset (fewer cells or genes)
  • Ensure you're not running other memory-intensive processes
  • Consider using a machine with more RAM
CUDA Errors

If you encounter CUDA errors:

  • Ensure you have a compatible GPU
  • Verify that CUDA is installed correctly
  • Check that the PyTorch version matches your CUDA version
  • Try running with CPU only by setting --device cpu
Luigi Task Failures

If Luigi tasks fail:

  • Check the logs in the logs/ directory for specific error messages
  • Verify that all dependencies are installed correctly
  • Ensure that the input files for the task exist
  • Try running the task with --local-scheduler for more detailed output
Weights & Biases Issues

If you encounter issues with Weights & Biases:

  • Verify that you're logged in with wandb login
  • Check your internet connection
  • Try running offline with --wandb offline
  • Check the W&B documentation for specific error messages

If you encounter other issues, please check the logs and consult the GitHub issues for the project.
